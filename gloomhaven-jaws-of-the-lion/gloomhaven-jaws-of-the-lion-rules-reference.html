1. Setup
1.1 Character Setup (Campaign Start Only)

There are four characters available: the Demolitionist, the Red Guard, the Hatchet, and the Voidwarden. Up to four players may each control one character. If you are playing alone, you will control two separate characters, which is a more complex experience; if there are two to four players, it is highly recommended that each player controls only a single character.

To set up your character, find the large and small boxes for your chosen character. Claim the character's figure and mat, the rectangular initiative order token from the character's zip bag, a hit point dial from the game box, the attack modifier deck (the deck of smaller cards bearing no warning label), and the first seven cards from the larger deck bearing no warning label. Of those seven cards, one is a player reference card and six are ability cards marked with an "A" in the crown under the title. The two decks with warning labels, the remainder of the large deck, the character sheet pad, and the strip of circular character tokens should remain in the box for now.

Place your character mat, player reference, six level "A" ability cards, attack modifier deck, and hit point dial in front of you. Set the red side of the hit point dial to the red number shown below the white "1" on the bottom of your character mat. The blue side of the dial is not used at this time. Place your initiative order token near the centre of the table.

Before beginning the campaign, each player takes one character sheet from their character's large box, writes a name in the name field, and marks the "1" next to "Level."

1.2 Scenario Setup Checklist

Perform the following steps before each scenario. Steps marked with a campaign note apply only from that point in the campaign onward.

  1. Open the Scenario Book to the correct scenario and place it in the middle of the table. Read the introduction text and special rules before playing.

  2. Consult the scenario key for the monsters and tokens required.

  3. Place each character's figure on a different starting hex icon in the Scenario Book, with no two characters on the same hex.

  4. Retrieve each required monster set: insert the stat card into a stat envelope with the correct level showing; place the standees into plastic stands with their numbers randomised; retrieve the initiative order token. Populate the map by consulting the rank indicator on each monster hex — use the row for the current player count (2, 3, or 4 characters). A black icon means no monster is placed; a white icon means place a normal monster (white stand); a gold icon means place an elite monster (gold stand). Set up only the monsters in the starting room; do not place monsters in unrevealed rooms.

  5. Place money tokens, trap tiles (with the appropriate number of damage tokens on top), and treasure tiles on the hexes depicted in the scenario.

  6. Retrieve the shared monster attack modifier deck (the 20-card "M" deck), place it next to the monster stat card, and shuffle it. Shuffle all character attack modifier decks.

  7. Place all character and monster initiative order tokens near the centre of the table.

  8. Retrieve and shuffle each monster set's ability deck and place it face down next to its stat card. Use the full eight-card ability deck for each monster set. (Scenario 2 onward.)

  9. Retrieve the battle goals deck, shuffle it, and deal two cards to each character in secret. Each character chooses one card to keep and discards the other face down without revealing or discussing it. (Scenario 4 onward.)

  10. Retrieve the element board and place all six element tokens in the Inert column. (Scenario 4 onward.)

  11. Set the scenario level (see §10) and rotate all monster stat envelopes to display the corresponding monster level.

When a supplemental map is required, the Scenario Book map layout graphic will show a supplemental page attached to the two-page spread with no dividing line between them. Open the Supplemental Scenario Book to the indicated page, fold it so only the single page is showing, and line it up next to the Scenario Book.

When monsters are spawned during a scenario as directed by special rules, place each spawned monster at its designated location or the nearest empty hex. Spawned monsters are treated exactly as if they had been set up when a room was revealed and always act in the round in which they are spawned.

2. Scenario Book Reference

Each scenario page contains the following labelled elements:

  • A — Scenario number, title, and city map board location, for ease of reference.
  • B — Goal of the scenario. When this is achieved, the scenario is successfully completed.
  • C — Scenario key, listing the monsters and tokens required to set up the map.
  • D — Map layout, indicating whether the Supplemental Scenario Book is required and how the pages are arranged.
  • E — Introduction text. Read before playing.
  • F — Special rules text. Read after the introduction; provides additional information not covered by the standard rules.
  • G — Conclusion text. Read only after the scenario has been successfully completed.
  • H — Rewards for successfully completing the scenario. Read after the conclusion.
  • I — Tips and rules reminders. Read as needed during play.
  • J — Obstacle designations, signified by a green border on the relevant hexes.
  • K — Wall designations. The thick, dark border lines surrounding each scenario's hex area are the walls of the scenario and cannot be crossed.

When the Supplemental Scenario Book is used only to display extra text that does not fit in the main two-page spread — rather than to provide additional map space — its page icon in the map layout graphic will be separated from the main spread by a dividing line.

Section breaks appear within some scenarios and indicate text that should be read only when a specific condition is met, such as when a door with the corresponding number has been opened or when another stated criterion has been satisfied. When a section break's condition is met, read it immediately and follow any special rules it provides. A scenario may contain more than one section break.

3. Round Structure

Gameplay within a scenario is broken into a series of rounds. Every round follows the same sequence:

  1. Card Selection
  2. Ordering of Initiative
  3. Character and Monster Turns
  4. End of Round

When the scenario goal is completed, play continues to the end of the current round before the scenario concludes. If the goal is completed on a character's turn, all remaining figures still take their turns for that round.

3.1 Card Selection

Each player simultaneously and privately selects two ability cards from their hand and places them face down in front of them. During this step, players may make general statements about their plans and discuss strategy, but they may not show other players their cards or provide specific information about any numerical value or card title.

Of the two cards chosen, one is designated as the initiative card, which determines the character's position in the initiative order for the round based on the initiative value shown in the centre of the card — lower numbers act earlier. The initiative card is placed face down beneath the other card, so that when both cards are flipped and revealed, the initiative card is on top and its value is visible.

Instead of selecting two ability cards, a character may declare that they will perform a long rest this round (see §8.3). This declaration is made during the card selection step.

3.2 Initiative Order

Once all players have selected their cards, flip each character's cards to reveal their initiative values. At the same time, flip the top card from each monster set's ability deck to determine that set's initiative and abilities for the round. Arrange all initiative order tokens from lowest to highest based on these values.

If there is a tie between a character and a monster, the character acts first. If there is a tie between two characters, the tiebreaker is the initiative value of their second card.

When a room is revealed mid-round and an ability card has not yet been flipped for that monster set this round, flip one immediately. If the revealed monsters' initiative is lower than the initiative of the character who opened the door, those monsters act immediately after that character's turn — place their initiative order token next in the order. Otherwise, place their token at the appropriate position in the normal initiative order.

3.3 Figure Turns

Beginning with the figure bearing the lowest initiative value, each figure takes a complete turn before the next figure acts. Characters take their turns as described in §5. Monsters take their turns as described in §6. When multiple monsters share the same initiative (all monsters belonging to the same set act on the same initiative), elite monsters act first in ascending standee number order, followed by normal monsters in ascending standee number order. Each monster completes its entire turn before the next monster acts.

3.4 End of Round

Once all figures have taken their turn, the round ends. Shuffle the discard pile back into any attack modifier deck that revealed a card bearing a shuffle icon during the round. If the element board is in use, move all element tokens one column to the left — from Strong to Waning, or from Waning to Inert. Tokens already in the Inert column remain there. Begin the next round with card selection.

4. Terrain and Map Features
4.1 Walls

The hexes of each scenario are contained within thick, dark border lines. These are the walls of the scenario; no figure may cross them. Walls also block line-of-sight (see §5.2).

4.2 Obstacles

Hexes with a green border are obstacles. Obstacles cannot be entered by any figure during normal movement and cannot be crossed. Unlike walls, obstacles do not block line-of-sight. Note that some obstacles occupy more than one hex, depending on how many hexes the green border surrounds. When an obstacle is destroyed by the Demolitionist (see §5.3), place destruction tiles on each hex of the destroyed obstacle to show those hexes are now normal traversable hexes.

4.3 Difficult Terrain

Hexes with a purple border are difficult terrain. A figure must spend 2 movement points simultaneously to enter a difficult terrain hex, rather than the usual 1. This additional cost is also taken into account when determining monster focus. Push and Pull forced movements are not affected by difficult terrain.

4.4 Traps

A hex containing a trap tile is a trap hex. When any figure enters a trap hex, the trap is sprung: the figure immediately suffers damage equal to the value shown on the damage token placed on the trap, and the trap tile is removed from the map. Traps are not obstacles. When determining focus and movement, all monsters treat trap hexes as obstacles unless moving through a trap is the only way they can reach a valid attack position; in that case, the monster finds the path that moves through the fewest traps possible.

4.5 Doors

Door hexes are designated by a blue border. Until a door is opened, it is treated as a wall for the purposes of line-of-sight and monster movement. A character may move onto a door hex as part of normal movement. When a character enters a door hex for the first time, the door is opened: place an activation token on the door hex to indicate it is open, then immediately set up all monsters in the newly revealed room. Monsters set up when a room is revealed always act in the round in which they were set up. The character then immediately resumes their turn, including any remaining movement points from the move ability that opened the door. Only monsters in the room where characters start are set up at the beginning of a scenario.

4.6 Objectives

Hexes with a yellow border are objectives. Objectives are treated as enemies by characters and can be targeted by attacks. However, objectives are unaffected by all negative conditions and by forced movement such as Push and Pull. No figure may move through an objective hex unless using the Jump effect. Objectives are not obstacles. An objective's maximum hit point value is defined by the special rules of the scenario, often using C, which represents the number of characters at the start of the scenario. Track damage suffered by an objective by placing damage tokens directly on it. When an objective's current hit point total is reduced to 0, it is destroyed: place a destruction tile on it to show that it is now a normal, traversable hex.

4.7 Empty Hexes

A hex is empty if it is within the walls of the scenario, is not difficult terrain, and does not contain a figure, obstacle, objective, or trap. Open doors and destroyed obstacles and objectives are all considered empty hexes, unless the hex also contains a figure or trap.

5. Character Turns

On a character's turn, the character performs one top action and one bottom action — one from each of their two played cards. A character cannot perform two top actions or two bottom actions. Either card may be used first and for either its top or bottom action. The character's choice of which card served as their initiative card no longer matters once the turn begins. An action — the entire half of a card — must be performed in the order written. Any individual ability within an action may be skipped. After the turn, both played cards are placed in the discard pile on the left side of the character mat, unless a card bears the lost icon, in which case it goes to the lost pile on the right side.

5.1 Movement Move

A Move X ability allows a figure to move up to X hexes. The figure does not have to use all available movement points. Figures can move through their allies — characters through characters and monsters through monsters — but movement can never end on an occupied hex containing another figure. Figures cannot move through enemies, obstacles, walls, or objectives.

Jump

If a move ability is accompanied by the Jump effect, each hex of that movement is unaffected by all terrain and by enemy figures. A jumping figure can move through obstacles and enemy figures; it does not spring traps while passing through trap hexes in motion, and it suffers no movement penalty for entering difficult terrain. Figures cannot jump through walls. Any movement using Jump must still end in a viable hex that contains no other figures or obstacles. If a Jump movement ends in a trap hex, the trap is still sprung.

Push and Pull

Push X and Pull X are forced movement effects that can be attached to attacks or exist as stand-alone abilities.

Push X forces the target to move up to X hexes away from the targeting figure. Each individual step of the movement must place the target spatially farther from the targeting figure.

Pull X forces the target to move up to X hexes toward the targeting figure. Each individual step of the movement must place the target spatially closer to the targeting figure.

Targets can be pushed or pulled through their allies, but not through their enemies, obstacles, or walls. If a target is pushed or pulled into a trap hex, the trap is sprung and the target suffers its effects. When a character performs a push or pull, they choose how far and in what direction the target is moved. When a monster performs a push or pull, players still choose the direction, but the target must be moved as far as possible. If there are no viable hexes into which the target can be pushed or pulled, the effect ends immediately. Forced movement from Push and Pull is not affected by difficult terrain.

5.2 Attacks Attack

An Attack X ability allows a figure to cause one targeted enemy within their range to suffer a base X amount of damage, modified by an attack modifier card. Figures cannot attack their allies unless otherwise specified. One attack modifier card is drawn from the attacker's deck for each separate target of an attack.

Melee

If no range value is written beneath an Attack, the attack is a melee attack and can only target adjacent enemies, unless otherwise specified.

Ranged

If Range X is written beneath an Attack, the attack is a ranged attack and can target any one enemy within X hexes of the attacker. Range cannot be counted through a wall. If a ranged attack targets an enemy adjacent to the attacker, that attack gains Disadvantage (see below).

Line-of-Sight

When any figure is targeted by an ability, the targeting figure must have a clear line-of-sight to the target in order to perform the ability. Line-of-sight is established if a straight line can be drawn from any part of the targeting figure's hex to any part of the target's hex without touching a wall line. Only walls block line-of-sight; obstacles and other figures do not.

Area Effects

Some attacks allow a figure to target enemies in multiple hexes. The targetable area is shown by a hex diagram using two icons.

A grey hex indicates the hex the active figure currently occupies. An area attack that includes a grey hex is always a melee attack.

A red hex indicates a position in which an enemy can be targeted by the ability. Area attacks, like all other attacks, cannot target allies.

If an area effect ability specifies a range and includes no grey hex, only one of the red hexes needs to be within the specified range, even if that hex does not contain an enemy. Any ranged area attack that targets an adjacent enemy still gains Disadvantage, and a separate attack modifier card must be drawn for each individual target. As with any targeted ability, figures can only target enemies in hexes they have line-of-sight to. Any rotation or mirroring of an area depiction is valid.

Target X

Some attacks are followed by Target X, meaning the attacker may target up to X different enemies within the attack's range. It is not possible to target the same enemy more than once from a single ability. A separate attack modifier card is drawn for each target.

Attack Modifier Cards

Any time a figure performs an attack, one card is drawn from the figure's attack modifier deck for each separate target. Apply the result as follows.

A numerical modifier is added to or subtracted from the attack value. A 2x card doubles the attack value. A null card causes the attack to deal no damage. The 2x and null cards each bear a shuffle icon: if either is drawn, the entire deck is shuffled at the end of the round.

Each character has their own attack modifier deck. All monsters collectively share one monster attack modifier deck.

Advantage and Disadvantage

If an attack has Advantage, the attacker draws two attack modifier cards and applies whichever result is better. If an attack has Disadvantage, the attacker draws two cards and applies whichever result is worse.

An attack cannot have multiple instances of Advantage or Disadvantage — they do not stack. If an attack has both Advantage and Disadvantage simultaneously, they cancel each other out and the attacker draws a single card as normal.

In any case of character Advantage where the choice between two drawn cards is ambiguous — for example when comparing a "+1 Wound" card against a "+2" card — the player may choose whichever card they prefer. In the case of ambiguous character Disadvantage, apply the first attack modifier card drawn.

Damage and Killing

After an attack is resolved and the final attack value has been calculated — modified first by any bonuses and then by the attack modifier card — the target suffers that amount of damage. When a monster suffers damage, place damage tokens equal to that amount in the stat envelope section corresponding to the number on the monster's standee. When the total of damage tokens in a monster's section equals or exceeds the monster's maximum hit point value, the monster is killed and removed from the map; all tokens in that section are returned to the tray.

5.3 Other Abilities Suffer Damage

Some abilities cause figures to suffer damage without an attack being made. This damage is not modified by attack modifier cards and is not affected by line-of-sight.

Heal

A Heal X ability allows a figure to cause one ally within the specified range — or themselves — to recover X hit points. Characters recover hit points by rotating the red side of the hit point dial upward. A figure cannot be healed above their maximum hit point value.

Experience

Some actions have an experience value attached to them, denoted by a star icon. When a character takes such an action, they gain the number of experience points shown. Experience gained during a scenario is tracked by rotating the experience wheel on the right side of the character's hit point dial. A character must still perform some part of the accompanying abilities in order to gain the experience — for example, a character cannot gain experience from an attack action if there is no eligible enemy to target. Basic actions (see §5.4) never trigger the experience value of the card used to perform them. If an action's experience is gated behind a specific written condition, that condition must be satisfied for the experience to be gained. Characters do not automatically gain experience from killing monsters.

Loot

Characters have two ways to collect money tokens and treasure tiles from the map. First, at the end of each character's turn, that character automatically loots the hex they currently occupy. Second, a Loot X ability allows the figure to loot their current hex and all hexes within range X of them.

When a money token or treasure tile is looted, move it from the map to the character's mat; it will be recorded at the end of the scenario. Money tokens and treasure tiles can only be looted during a scenario — any tokens remaining on the map at the scenario's end are not collected.

When a monster dies, one money token is placed in the hex in which it died, where it may be looted by a character.

When a character collects a treasure tile, remove it from the map, note the number shown on the tile's icon, and refer to that entry in the Treasure Index in the Rules Glossary. The looting character immediately receives the listed reward.

Active Bonuses

Some actions give a character bonuses that last until the end of the round, the end of the scenario, or until a certain number of uses are exhausted. When such an action is played, place the card in the character's active area above their character mat.

Persistent bonuses are active from the time the action is played until either the end of the scenario or their allotted uses run out. When placing a persistent bonus card in the active area, place one character token on the first use slot. Advance the token one position — moving left to right, top to bottom — each time the effect is triggered. When the token moves off a use slot bearing an experience icon, the character gains that experience. When the token moves off the last use slot, remove the card from the active area and place it in the lost or discard pile depending on whether the action bears the lost icon. A character must activate a persistent bonus and advance the token every time the effect is triggered, even if no benefit is gained.

Round bonuses are active from the time the action is played until the end of the round, at which point the card is placed in the lost or discard pile depending on the lost icon.

Either type of active bonus card may be moved voluntarily to the lost or discard pile at any time before its duration expires; however, doing so immediately removes all bonuses and effects that card was providing.

Shield

Shield X is an active bonus. When a figure with Shield X suffers damage from an attack, they reduce that damage by X, to a minimum of 0. This reduction is applied after attack bonuses and the attack modifier card have been factored in. For instance, if an "Attack 2" is performed against a figure with "Shield 1" and a +2 modifier card is drawn, the modifier is applied first, bringing the attack value to 4, and then the shield is applied, so the figure suffers 3 damage. Multiple Shield bonuses stack and may be applied in any order. Shield only reduces damage suffered from attacks — it does not reduce damage from traps or any other non-attack source.

Granting Actions

Some abilities grant actions to other figures — both allies and enemies. The initial text of such an ability specifies who receives the granted action as a targeted effect; the granted action is then described, and all normal rules for attacking and moving apply. If a figure is granted an attack action, it uses its own attack modifier deck unless otherwise specified. A figure granted an action does not take an additional separate turn — it performs the action during the granting figure's turn.

Self

Some abilities specify the target as Self. This means the figure using the ability is the only possible target.

Destroying Obstacles

The Demolitionist has abilities that cause an obstacle within a specified range to be destroyed, making the hex or hexes the obstacle occupied traversable. When an obstacle is destroyed, place destruction tiles on each hex of the destroyed obstacle — including any hexes that may be out of range of the ability — to show that those hexes are now normal, traversable hexes.

5.4 Basic Actions

Instead of performing the printed action on a card, any card may be used to perform a basic action: Attack 2 as a top action, or Move 2 as a bottom action. When a card is used for a basic action, it is always placed in the discard pile regardless of any lost icon on that card. Basic actions never trigger the experience value of the card used to perform them.

5.5 Elements

Some actions have an elemental affinity associated with them, represented by one of six element icons: Fire, Ice, Air, Earth, Light, or Dark.

Infusing an element: If an element icon appears by itself within an action's text, performing any part of that action requires the figure to infuse the battlefield with that element. At the end of the turn in which the action is performed, move the corresponding element token to the Strong column of the element board. It is not possible to infuse an element and then consume it on the same turn, because the infusion does not take effect until the end of the turn.

Consuming an element: If an element icon appears covered by a consume symbol, followed by a colon and a description, the figure may consume that element to augment the ability. If the corresponding element token is in the Strong or Waning column, the figure may move it to Inert to gain the described augment. If an action contains multiple separate augments, the character may choose which ones to apply.

Wild icon: A multicoloured wild icon represents any one of the six elements. If a wild icon appears as an infuse effect, the players choose which element is infused. If it appears as a consume requirement, the players choose which element is consumed.

Monsters and elements: Monsters also have abilities that infuse and consume elements, and the same rules apply. Monsters always consume elements if they are able to do so. When a monster set consumes an element, every activated monster of that set gains the benefit — not only the first monster to act that round.

5.6 Negating Damage with Lost Cards

Any time a character suffers damage from any source, the character may choose to negate the entirety of that damage by losing cards instead of reducing their hit point dial. To negate a single source of damage, the character may either lose one card from their hand or lose two cards from their discard pile. Lost cards are immediately moved to the character's lost pile. Note that the two cards a character has selected to play for the current round are not in their hand during that round — they move to the discard pile during the character's turn. Losing cards to negate damage reduces the number of rounds the character can play before becoming exhausted, but it can be preferable to taking damage that would otherwise reduce the character's hit points to 0.

5.7 Character Damage and Exhaustion

Any time a character suffers damage, reduce their current hit point value by that amount by rotating their hit point dial. If a character's hit point value ever reaches 0, they are immediately exhausted: remove the character's figure from the map. An exhausted character can no longer participate in the scenario in any way. If all characters become exhausted, the scenario is immediately lost.

Exhaustion is not permanent — all characters are fully refreshed at the end of every scenario, even those who became exhausted.

6. Monster Turns
6.1 Monster Ability Cards

From Scenario 2 onward, each monster set acts according to a card flipped from its ability deck during the initiative order step (see §3.2). One card is flipped per monster set — all monsters of the same set act on the same initiative and use the same abilities for the round. If no monsters of a set are currently on the map, do not flip a card for that set.

Monster ability cards provide value modifications to the monster's base stats, rather than absolute values. For example, if a card reads "Attack +1" and the monster's base attack value is 2, the monster performs an Attack 3 that round. A monster only performs the abilities listed on its ability card — if the card does not include a Move ability, the monster does not move. If any ability card bears a shuffle icon, all played cards from that deck are shuffled back into the deck at the end of the round.

6.2 Monster Focus

Before a monster does anything else on its turn, it determines a focus: the character it will attempt to attack. The focus is the character the monster can perform its attack against using the least amount of movement.

For a monster performing a melee attack, focus is determined by finding the shortest path to a hex adjacent to a character. For a monster performing a ranged attack, focus is determined by finding the shortest path to a hex from which it can attack a character within the attack's range.

Line-of-sight is not required to establish focus. It does not matter whether the monster can actually reach the end of its path this turn — so long as a path exists given infinite movement, the monster will still focus on that character.

If multiple characters can be reached in the same number of movement steps, the monster focuses on the character who appears earlier in the current initiative order.

If a character is already adjacent to the monster (for melee) or within range (for ranged), that character becomes the focus without the monster needing to evaluate a path.

If no valid path exists to reach any character at all, the monster has no focus and will neither move nor attack this turn.

6.3 Multi-Target Monster Focus

If a monster can attack multiple targets on its turn, it first finds its primary focus using the normal rules. It then seeks additional foci by finding the position from which it can attack its primary focus and the greatest number of additional targets simultaneously, using the minimum required movement to reach that position.

There may be times when it is unclear how a monster should behave. In those cases, do your best to have the monster behave intelligently and do not worry too much about the specifics.

6.4 Monster Movement

Once a monster has established its focus, it moves toward that focus using the minimum distance necessary, up to its base Move value as modified by its ability card. If the monster is already in position to attack its focus, it does not move. If the monster cannot get any closer to its focus because another monster is blocking its path, it also does not move.

Ranged monsters attempt to avoid adjacency to their focus, since performing a ranged attack on an adjacent target gains Disadvantage. If a ranged monster is adjacent to its focus and is able to move far enough to no longer be adjacent, it will do so. However, if the monster already has Muddle or another source of Disadvantage, it would gain no benefit from moving away — it cannot reduce its Disadvantage further — and so it does not move for this reason. A monster performing a melee attack, or not performing any attack (due to Disarm or the absence of an attack ability on its card), moves normally to get adjacent to its focus.

If it is ever ambiguous which path a monster should take because multiple options are equally viable, the players decide which path the monster takes.

6.5 Monster Attack

Once a monster has moved, if it is now in a valid position to attack its focus, it does so. The monster uses its base Attack value as modified by the bonus on its ability card, and draws one card from the monster attack modifier deck per target. The targeted character or figure then suffers the resulting amount of damage.

6.6 Monster Active Bonuses

Some monster stat cards show permanent active bonuses displayed to the left of the normal monster stats and to the right of the elite monster stats. These bonuses are always applied regardless of what the monster's ability card dictates. If the bonus is a negative condition, that condition is applied to all of the monster's attacks.

6.7 Monster Stat Cards

The monster stat card is inserted into a stat envelope so that only the stats for the current scenario level are visible. Rotating the card within the envelope changes the displayed level. The stat card shows the monster's name and current level, normal (white) and elite (gold) stat columns, the monster's maximum hit point value, its base Move value, and its base Attack value.

Damage suffered by a monster is tracked by placing damage tokens in the section of the stat envelope that corresponds to the number on that monster's standee. When the accumulated damage tokens in a section equal or exceed the monster's maximum hit point value, the monster is killed and removed from the map; all tokens in that section are returned to the tray.

6.8 Boss Mechanics

All bosses use the universal Boss ability deck, which consists primarily of cards labelled "Special 1" or "Special 2." These special abilities are described on the boss's own stat card. A boss's stats are often expressed using C, which stands for the number of characters in the current scenario.

Bosses are immune to certain negative conditions; the specific immunities are listed on the left side of the stat card, beneath the level indicator.

A boss with "—" for its base Move value cannot move voluntarily, even if a Boss ability card would otherwise provide a Move ability. Such a boss can still be moved by Push, Pull, and granted actions.

Some bosses have special abilities too detailed to summarise on the stat card. In these cases, the full description is found in the scenario's special rules.

7. Conditions

Conditions may be applied as a stand-alone ability or as an effect attached to an attack. Even if an attack deals no damage, any conditions it specifies are still applied. If a condition is applied to a character, place the corresponding token in the lower-left section of their character mat. If a condition is applied to a monster, place the token in the stat envelope section corresponding to the number on that monster's standee.

Standard conditions (Immobilize, Stun, Muddle, Strengthen, Disarm) remain on the target until the end of the target's next full turn. This may occur in the same round if the target has not yet acted this round, or in the following round if they have already acted.

Poison and Wound persist until removed by a Heal ability. Bless and Curse persist in the attack modifier deck until the relevant card is drawn. All Bless and Curse cards are removed from every attack modifier deck at the end of every scenario. A figure cannot have multiple simultaneous instances of Poison or Wound.

7.1 Negative Conditions
Condition Effect
Immobilize The figure cannot perform any Move abilities.
Stun The figure cannot perform any abilities, including Move and Attack.
Muddle All of the figure's attacks gain Disadvantage.
Disarm The figure cannot perform any Attack abilities.
Poison All attacks targeting this figure gain a +1 attack bonus applied before the attack modifier card. This condition is removed only when the figure is affected by a Heal, but the Heal does not restore any hit points while Poison is active.
Wound The figure suffers 1 damage at the start of each of their turns. This condition is removed only when the figure is affected by a Heal; the Heal still restores hit points unless the figure also has Poison, in which case both conditions are removed and the Heal has no further effect.
Curse Shuffle one Curse card into the figure's attack modifier deck. The card remains in the deck until it is drawn, at which point it is removed. A maximum of 10 Curse cards may be present in any one attack modifier deck at a time. Curse cards for the monster deck are marked with an "m"; Curse cards for character decks are marked with a "". When drawn, a Curse card functions as a null card*.

8. Card Management
8.1 Discard and Lost Piles

After a character uses a card, it goes to one of two piles on the character mat. Cards without the lost icon go to the discard pile on the left side of the mat. Cards bearing the lost icon go immediately to the lost pile on the right side of the mat. Cards in the lost pile cannot be returned to the character's hand for the remainder of the scenario. The two cards a character has selected to play for the current round are not in their hand during that round — they move to the appropriate pile during the character's turn.

8.2 Short Rest

If a character has at least two cards in their discard pile, they may perform a short rest at the end of any round. To short rest, the character takes all cards in their discard pile, shuffles them, and randomly places one face down in their lost pile. The remaining cards are returned to the character's hand and the next round begins as normal.

If the character would prefer to keep the card that was randomly selected to be lost, they may instead suffer 1 damage and randomly lose a different card from the discard pile. This exchange can only be made once per short rest.

8.3 Long Rest

Instead of selecting two ability cards at the start of a round, a character may declare that they will perform a long rest. A long-resting character acts on initiative 99 — after virtually all other figures. On their turn, the character chooses one card from their discard pile to transfer to their lost pile, then returns all remaining discard cards to their hand. The character also performs a Heal 2, Self action and uprights all spent items (those rotated 90° to show they have been used), refreshing them for use again. Monsters and other figures still act normally during a round in which a character is long resting, and the resting character may be attacked.

8.4 Exhaustion from Card Loss

If a character is unable to play two ability cards at the start of a round because they do not have at least two cards in their hand, that character is immediately exhausted (see §5.7). This can occur when too many cards have been moved to the lost pile through resting or through the use of actions bearing the lost icon.

9. Between Scenarios
9.1 End of Scenario — All Outcomes

Whether the scenario was won or lost, all characters fully refresh before the next scenario or session:

  • Recover all ability cards — return the full hand, discard pile, and lost pile to the hand.
  • Remove all condition tokens from all character mats.
  • Return each character's hit point dial to their maximum hit point value, which is the white number shown beneath the character's current level on the character mat.
  • Refresh all items: return all consumed items (flipped face down) to face-up, and return all spent items (rotated 90°) to upright.
  • Remove all Bless and Curse cards from all attack modifier decks.

To save the game between sessions, place each character's character sheet, attack modifier deck, and current hand of ability cards into their zip bag along with the initiative order token, then return that zip bag and the character mat to the large character box. With the city map board updated and all character materials stored, the campaign can be resumed easily at any time.

9.2 Scenario Success — Post-Scenario Steps

After successfully completing a scenario, perform the following steps:

  1. Read the conclusion text and rewards in the Scenario Book.

  2. Mark the scenario as complete on the city map board by filling in the white box on its sticker. Completed scenarios cannot be replayed.

  3. If the rewards list a New Location, place the corresponding sticker on the city map board in the indicated position.

  4. Record gold: each character counts the money tokens they collected during the scenario, converts them to gold at the current rate (see §10 for the gold per token at each scenario level), adds any flat gold rewards from the scenario, records the revised total in the gold notes of their character sheet, and returns the money tokens to the tray. Characters cannot share gold or money tokens at any time.

  5. Record experience: each character records their total experience gained during the scenario (from the experience dial) plus the scenario completion bonus (see §10 for the bonus XP at each scenario level). Record this in the XP notes of the character sheet.

  6. Check whether any character has reached the XP threshold for the next level and perform leveling up if so (see §9.5). Characters never lose experience and do not reset their experience total when leveling up.

  7. Award any item rewards: some rewards go to only a single character; the players collectively decide which character receives it.

  8. Resolve battle goal results: each character who met the criteria of their chosen battle goal during the scenario gains 1 check mark. On a failed scenario, no check marks are awarded. Discard all battle goal cards whether or not the goal was met.

  9. For every 3 check marks a character has accumulated (tracked in the lower-right area of the character sheet, maximum 18 total), the character earns one perk (see §9.6).

  10. Encounter a city event (see §9.7).

  11. Buy, sell, or trade items (see §9.4).

Card pool advancement: At certain points in the campaign, characters expand their pool of available ability cards.

After Scenario 1, each character retrieves their two level "B" cards from their large box and swaps them for the two level "A" cards of the same name, so that each character still has six cards in their available pool. The two "A" cards are permanently retired and will never be used again.

After Scenario 2, each character retrieves the remainder of their level "1" cards from their large box and adds them to their existing two level "1" cards, then permanently removes all level "A" and level "B" cards. The full set of level "1" cards is now available for the rest of the campaign. Each character's pool of available cards now differs in size depending on their hand limit, shown in the upper right of the character mat.

After Scenario 4, each character retrieves their three level "X" cards from their large box and adds them to their pool of available ability cards. These cards are available for the rest of the campaign and are intended to give more options and flexibility going into a scenario. Characters are still only ever allowed to bring into a scenario a number of cards equal to their hand limit — with more cards now in the pool than the hand limit allows, players must choose which cards to leave behind at the start of each scenario. This decision can be made after reading the upcoming scenario's information. If you are not ready to make those decisions, leave behind the "X" cards; they are generally more complicated and situational than level "1" cards, but under the right circumstances can be very useful.

9.3 Scenario Failure

If all characters become exhausted before the scenario goal is achieved, the scenario is failed and must be replayed from the beginning. When resetting a failed scenario, carry over the following into the next attempt: all gold from money tokens collected during the failed attempt, all experience gained, and all rewards received from looting treasure tiles. Treasure tiles, once looted, can only be claimed once per campaign — they do not reset. Everything else in the scenario is reset to its starting state.

If a scenario is proving consistently difficult, first confirm that no rules are being overlooked. If the scenario is being played correctly and remains too challenging, you may at the start of any attempt rotate all monster stat cards in their envelopes to the level "0" section to reduce the difficulty.

9.4 Items

Available items are those in the item deck within the currently unlocked index range. Items outside that range are placed on the "Unavailable Items" side of the card divider in the game box; available items are on the other side and may be purchased.

Buying: A character pays the gold value shown on an item card to transfer it from the shop supply to their possession. A character can never own more than one copy of the same item.

Selling: A character may sell an item they own back to the shop for half its gold value, rounded up, which is added to their total gold.

Trading: Players may freely transfer items between characters at any point between scenarios. Gold cannot be traded.

Each item card shows the item's gold value, item type (Head, Body, Feet, Hands, or Small), and item function describing when and how it can be used. Items generally provide additional abilities or bonuses a character can use on their turn. Using an item is free and can be done in addition to the two actions performed from ability cards. The quantity shown on the card indicates how many copies of the item exist in the game.

The item usage icon in the lower-right corner of the card indicates how often the item can be used in a scenario.

A consumed item can be used once per scenario. Once used, flip the card face down to show it has been consumed.

A spent item can be used once per long rest. Once used, rotate the card 90° to show it has been spent. Spent items are refreshed when the character performs a long rest.

A persistent item has no usage icon and can be used whenever the situation described by its function applies.

No item is permanently removed from a character when used — all items are refreshed at the end of every scenario.

Characters are not limited in the total number of items they can own, but they may only bring a limited set into a scenario: 1 Head item, 1 Body item, 1 Feet item, 2 Hand items, and a number of Small items equal to half their level rounded up (1 Small item at levels 1–2, 2 at levels 3–4, and so on).

9.5 Leveling Up

After any scenario, if a character's total experience equals or exceeds the threshold for the next level shown on their character sheet, the character advances to that level and immediately performs all of the following steps:

  1. Choose one new ability card from those at or below the new level and add it to the character's pool of available ability cards. For example, when reaching level 2, choose one of the two available level "2" cards. When reaching level 3, choose one card from either the two level "3" cards or the level "2" card not chosen previously.

  2. Gain one perk (see §9.6).

  3. Increase maximum hit points: the new maximum is the white number shown beneath the new level number on the character mat.

Characters never reset their experience total upon leveling up and can never lose experience.

When a character reaches level 5, they open the second pack of ability cards in their character's large box, which contains their higher-level cards and includes special instructions.

9.6 Perks

Each time a character gains a perk — whether through leveling up, accumulating check marks, or a scenario reward — the player chooses any one unchecked perk from the list on the right side of their character sheet, checks the box next to it, and immediately applies that modification to their attack modifier deck. Modifications may include removing unfavourable cards from the deck or adding new, character-specific cards drawn from the separate deck of character attack modifier cards in the character's box (marked with a "Halt!" warning). All perk modifications are permanent — choose carefully.

Check marks are tracked in the lower-right area of the character sheet, below the perks. A character may accumulate a maximum of 18 check marks in total. One perk is earned for every 3 check marks gained.

9.7 City Events

After each successfully completed scenario, the party encounters a city event. Before the first city event of a campaign, retrieve the events deck from the game box and shuffle it.

Draw the top card and read the introductory text. The card presents two options; the players collectively choose one. Some options are only available if specific requirements are met — for example, a particular character being present. Once — and only once — an option is chosen, flip the card over and read the corresponding outcome (A or B). This consists of additional flavour text followed by specific effects, all of which must be applied if possible.

After the event is resolved, place the card on the "Encountered Events" side of the card divider in the game box; it is removed from the game permanently. Return the remainder of the deck to the "New Events" side.

10. Scenario Level Reference

The scenario level governs monster difficulty and determines trap damage, gold conversion, and completion experience for the scenario. The recommended (Normal) scenario level is the average level of all participating characters divided by 2, rounded up.

Players may freely adjust difficulty: Easy is Normal − 1; Hard is Normal + 1; Very Hard is Normal + 2.

The monster level used in a scenario — the stat group shown in the stat envelope — is always equal to the scenario level.

11. Miscellaneous Rules

Player count changes mid-campaign. If a player cannot attend a session or an extra player wants to join, characters may be added or removed as needed. The game accommodates two to four characters. A newly added character begins at level 1 with all of their level "1" and "X" cards. The scenario level scales to the average level of the characters currently participating, so it is fine if a character falls behind in level or if a new character begins at level 1 while others are at a higher level.

Monster standees run out during setup. If there are not enough standees in a set to place all required monsters, set up what is available starting with elite monsters and do not set up the rest.

Monster Loot abilities. When a monster uses a Loot ability to collect a money token or treasure tile, return it to the tray — it has no effect on the monster. If a treasure tile is looted by a monster, it can only be recovered by replaying the scenario from the beginning.

Closing scenarios. Some scenario rewards instruct the party to close specific scenarios, which means marking them off on the city map board so that they can no longer be played. Not all scenarios will be played as part of a campaign — sometimes choices must be made.

Restarting the campaign. The city map board is difficult to reverse, but all card decks can be reset. Each card has a white number at its bottom; arrange all decks in ascending order by this number to restore them. Obtain fresh character sheets for each character to begin again.

Monster behaviour. If you are uncertain how a monster should behave in a particular situation, an extensive appendix with illustrated examples of monster behaviour is available in the Rules Glossary.
